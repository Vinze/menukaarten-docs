    Conventions

    Files

    To ensure a good working of the different framework parts it is necessary to code by some predefined conventions. All file names are lowercase, and words are separated by an underscore. For example: config.php, user_settings.php, base_model.php, form_helper.php.

    Models

    Class names are written in CamelCase and underscore, end with _Model, and are singular. Functions are lowercase and also use an underscore.

    Example: user_model.php 
    class User_Model extends Base_Model {
	public function find_by_id($id) {
	}
}
    Example: category_model.php 
    class Category_Model extends Base_Model {
		public function save($data, $id) {
	}
}

    Controllers

    Class names are in CamelCase and underscrore, have to end with _Controller, and are written in plural. All functions are in lowercase and separated with an underscore, and have to start with action_.

    Example: users_controller.php 
    class Users_Controller extends Base_Controller {
	public function action_index() {
	}
	public function action_edit() {
	}
}

    Views

    Views files are placed in the main views directory in a subdirectory named after the controller which uses them. All files need to be lowercase and nicely grouped together. Some examples:

    users/index.php
users/new.php
cms/admin/index.php
